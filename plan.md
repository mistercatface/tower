After PR 3, the fracture port is a geometry + spawn + draw track, not more solver work — PR 1–3 are the sim foundation. Keep circles as CircleShape on the hot path (balls, small debris, anything round); only props with a real non-circular footprint get the polygon pipeline. Pull the generic half out of BOIDS PRIMITIVES.js — ear-clip triangulation, edge subdivision into poxels, neighbor graph, splitPolygon on impact — into something like Libraries/Spatial/collision/polygonFracture.js (or colocated with splittable props), wired to WorldProp death/split instead of partitionCrateLocal rectangles. Each connected poxel component spawns as a pushable shard with a PolygonShape, mass from area, and inherited velocity; pushableContactSolver + SatCollision already sim them with no circle fakery. Rendering is the other half: a vertex-keyed PropDrawRecipe through QuantizedSpriteCache (extruded pseudo-3D crate faces per shard, not flat BOIDS fills). Authoring stays generic — asset or editor supplies a vertex outline, fracture runtime does the rest; concave/convex all triangulate the same way. Skip BOIDS’s convertCircleToPolygon / 8-gon path entirely; optionally add a terminal sliver cap (despawn or static decal) so triangle counts don’t explode. PR 3 first because friction + warm-start is what makes dense polygon piles settle instead of ice-skating.
