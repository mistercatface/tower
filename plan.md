Paragraph 1 — Shape-first props (done; do not regress). Physics and draw both read `body.shape`: circles are `CircleShape(radius)`, everything else is a convex `PolygonShape` built from asset `localFootprint` or `syncCollisionShape` (flipper, pipe). Mass, inertia, broadphase, sprite cache keys, and vector overlay all derive from that shape — no parallel `halfExtents`, `collisionShape`, or rect draw paths on world props. Kinetic boxes (`crate`, `box_2x4`, `custom_box`), proof shapes (`tri_wedge`, `hex_block`), and balls share one spawn path (`initWorldPropShape`); 3D draw goes through `createPolygonPrimitive` → `drawExtrudedConvexPolygon` (or sphere mesh for balls) into the cached prop bake. Flipper and pipe stay bespoke mesh visuals with polygon collision — that is intentional. Anything new that looks like a solid prop must enter through this pipeline before it touches breaking.

Paragraph 2 — Convex interaction parity (current work). SAT + `kineticContactSolver` already handle circle–circle, poly–poly, and circle–poly; the gap is confidence and coverage, not missing math. Make arbitrary convex polygons behave as cleanly as balls before any fracture: poly–poly contact tests with asserted normals and post-solve separation (not just “didn’t tunnel”), polygon wall resolution without circle-radius shortcuts on long/thin footprints, sleep/wake and friction parity on `tri_wedge` / `hex_block` / `box_2x4` stacks, and flipper/pipe left on their sync-collision path. Concave silhouettes and multi-part bodies can wait — if you need an L-shape, decompose into 2–3 convex polygons on one entity first. Do not reintroduce fracture until stacking, sliding into walls, and resting contact all look boring on the existing proof props.

Paragraph 3 — Poxel fracture as shape mutation (endgame). Opt-in per asset (`fracture: true` on splittable crates/boxes). One fixed world-space minimum edge length (`POXEL_TARGET_EDGE`, never scaled by parent spawn size): any footprint — spawned 16×16 or a fragment of the same type — runs the same bake: outline → triangulate → subdivide to target edge → `parts[]` for draw adjacency and split topology. Collision stays one convex `PolygonShape` per body (outline or union of surviving parts); on high-impulse impact, peel/flood connected triangle components — largest piece updates the parent shape in place and re-syncs mass/inertia; smaller pieces spawn as the **same prop type** through the identical shape init path so a spawned crate and a crate fragment are indistinguishable sim objects. Repeat breaks until no edge exceeds the minimum — that is the floor, not a separate “shard” entity or hidden rect draw. Visuals are per-triangle stroke/fill on the polygon recipe from paragraph 1; fracture only mutates `shape` + `parts`, never parallel boxes or parent-size peel radii. Old splittable/poxel code in Deprecated is reference only — re-port on top of shape-first wiring, not beside it.
