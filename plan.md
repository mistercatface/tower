PR 1 — Overlay command pipeline (world-only, no more stray ctx). Introduce a single world-attached overlay path: features emit typed commands (segment, polyline, arrow, ring, aabb, etc. in world units at world anchors), and one executor in Libraries/Render/overlays/ turns them into canvas draws under viewport.apply — no getCanvasLineScale, no /zoom hacks, no screen-stable branch. Migrate everything currently scattered across drawActivePathOverlay, drawSandboxSelection, marquee/place-preview, drag-launch rubber bands, button/corridor wire overlays, and the controller?.draw*() sprawl in preview.js into collect*OverlayCommands + one simulation effect pass. Delete the accidental screen-stable convention from gameplay-facing canvas draws; line widths and arrow sizes are world constants that scale naturally with zoom. Review bar: sandbox/editor feedback never touches ctx directly — only overlay commands or existing prop sprite recipes.

PR 2 — Quantized bake + blit for interface geometry. Extend the prop cache pattern to overlays: each command type gets a small OverlayDrawRecipe, a GRID_STAMP_RENDER_KEY/OVERLAY_RENDER_KEY, and keys bucketed by viewer-relative quantize (quantizeViewOffset), zoom step, topology, and dynamic state (selection, nav mode, anim frame) — same contract as buildPropSpriteKey / drawCachedPropSprite. Static-ish glyphs (selection ring, path arrowhead, flow steering arrow, destination marker) bake once per bucket and blit anchored at the world pose; only truly live geometry (marquee drag rect, in-progress wire) draws immediate world-space through the executor. Collapse duplicate immediate-mode paths as recipes land. End state: adding a new input feedback visual means a recipe + key buckets, not a new zoom-sensitive draw function.

PR 3 — Camera-relative pseudo-3D for props and overlays. Audit every world prop draw recipe and overlay anchor so geometry is authored in prop-local 3D and projected with projectPropVertex / elevationCameraFromViewer at bake and blit time — balls, boxes, selection rings, path arrows, and drag-launch bands originate from the projected foot/surface, not flat (prop.x, prop.y) circles on the ground plane. Props that still bake as screen-aligned 2D stamps get mesh or extruded-silhouette recipes with proper viewer-dependent keys; overlays inherit the same anchor (prop proxy + projected offset) so a selection ring hugs a ball’s apparent outline when the camera swings. Wire overlay command collection to pass px/py/zoom into cache lookup so baked interface sprites parallax consistently with WorldSceneRenderer props. End state: one visual language — everything on the main canvas is world-anchored, camera-relative, and cache-friendly; no flat HUD pretending to live in the scene.
