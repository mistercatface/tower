PR 1 — Segment2D: shared segment intersection
Expand Libraries/Math/Segment2D.js into the canonical home for 2D segment math by promoting the private segmentsIntersect from WallGeometry.js, adding a segmentIntersectionPoint (or equivalent) that returns the hit point plus t/u parameters when segments overlap, and moving distanceSegmentToSegment out of WallGeometry into Segment2D. Update WallGeometry (and only its direct segment-intersection callers) to import these instead of owning copies. This PR is intentionally narrow: no behavior changes, just one shared module and one consumer rewired so pointer, path, and wall code can all reach for the same segment primitives.

PR 2 — Poly2D: scalar rotation + point-in-polygon
Add the comfy transform scalars to Libraries/Math/Poly2D.js — rotateXYInto / rotateXY for precomputed cos/sin rotation, and transformPoint2DInto for rotate-local-then-translate (what rotatePoint already does by hand). Refactor rotatePoint and rectCorners to call those helpers so the API is dogfooded before anyone else uses it. In the same PR, add pointInPolygon (ray-cast or winding number — pick one, document on-edge behavior) taking a flat {x,y}[] or scalar coordinate list matching existing poly conventions. No new folder, no Bezier yet; this PR is pure Math library surface with internal refactors only.

PR 3 — Dedupe rotation at call sites
Mechanical sweep replacing scattered inline x _ cos - y _ sin blocks with the new Poly2D/Vec2 scalars at the highest-duplication sites: SatCollision.js, Spatial/geometry/circleSweep.js, kinematics ragdoll code, prop geometry (pipeElbowGeometry, flipperPaddle, etc.), and any other files grep turns up. Each change should be a straight substitution — same cos/sin precomputation, same out-param vs alloc patterns as the surrounding code — with no logic changes. This PR should be easy to review as a diff-of-imports-and-one-liners, and it’s where you actually feel the library get comfier rather than just bigger.
