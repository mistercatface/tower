Paragraph 1 — Lock down the physics contract before adding features. Treat body.shape as the single source of truth: a circle is CircleShape(radius); anything with 3+ vertices is PolygonShape(localVerts) rotated by facing. Stop letting halfExtents, collisionShape: "box", and drawBox be parallel definitions of the same thing. WorldProp construction should branch once: circle path (what pool balls already do) or polygon path (local vertex list in body space, synced to PolygonShape). Mass, inertia, broadphase radius, and vector overlay all derive from that shape — not from an AABB side channel. Crates can stay visually wooden boxes for now, but their collision footprint should already be the polygon, even if it's still 4 corners. This step doesn't add fracture, doesn't add poxels, doesn't change gameplay. It just makes "3+ vertices" a real first-class body type the solver already mostly supports.

Paragraph 2 — Make draw follow shape, not the other way around. Right now kinematic vector mode can swap circle/rect silhouettes, but default 3D draw has no polygon recipe — boxPrimitive ignores footprintVertices and poxels entirely. Add one cached draw path per shape class: circle (exists), convex polygon (fill outer loop + optional edge stroke, extruded to prop height like you do walls-lite). Wire it through PropDrawRecipe → drawCachedPropSprite so topology and size bucket the cache key. Extend vector mode with { kind: "polygon", vertices } so debug view matches sim. A triangle prop, a hex prop, and a crate should all use the same polygon draw/collision pipeline; crate is just an asset whose footprint happens to be a rectangle and whose recipe adds plank decoration on top. Until this lands, do not touch fracture — you will keep "simulating triangles, drawing rectangles."

Paragraph 3 — Prove mixed-shape interaction before concavity or breaking. You already have circle–circle, poly–poly, and circle–poly in SatCollision plus impulse solving in kineticContactSolver. The gaps are everything around that: wall resolution is still circle-biased, inertia for arbitrary polygons is approximated from halfExtents, and props like logs/stand-tip have custom collision sync that bypasses the generic path. Spend this phase making arbitrary convex polygons behave as cleanly as balls: spawn test shapes (triangle, pentagon, long bar), stack them, slide them into walls, verify sleep/wake, friction, and vector overlay parity. Add tests that assert contact normals and separation for poly pairs, not just "didn't tunnel." Concave silhouettes can wait — decompose into 2–3 convex PolygonShape parts on one entity if you need an L-shape, which is how real engines start before full concave contact.

Paragraph 4 — Only then add fixed-size poxel fracture as a shape mutation, not a new entity species. Define one constant world-space poxel edge length (your POXEL_TARGET_EDGE, never scaled by parent spawn size). Any convex footprint — spawned at 8×8 or grown by repeated breaks — goes through the same pipeline: outline → triangulate → subdivide to target edge → store as parts[] for draw and adjacency for split. On impact, peel/flood connected components; largest updates the parent's shape in place; others spawn as the same prop type with geometry run through the same bake path so a 16×16 spawned crate and a 16×16 fragment are identical sim objects. Fracture mutates shape + draw parts; it does not spawn crate_shard boxes, does not stash triangles in poxels while drawing halfExtents, and does not use parent-size peel radii. Visual tessellation is stroke-each-triangle on the polygon recipe from paragraph 2 — that is the whole reason you would see triangles at all.

The jump that burned you was doing paragraph 4 while paragraphs 1–2 were still hollow: fracture math without polygon draw or shape-first wiring will always look like "full crate plus bonus boxes." If you want a single next commit after the reset, do paragraph 1 + the minimal polygon draw stub from paragraph 2 — one non-rect convex test prop you can see and collide with. Everything else waits on that looking boring and correct.
