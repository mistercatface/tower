PR 1 — Sleep and active-set policy. Today almost every kinetic body in the stress demo stays in \_activeKineticBodies because broadphase overlap with any neighbor blocks sleep, so a settled pile of 500 balls still runs full neighbor queries and pair gates every frame. This PR defines what “awake” actually means: per-body sleep eligibility, rules for waking on impulse/contact/goal change, and when mutually resting contact neighbors are allowed to sleep together instead of vetoing each other forever. Wire syncActiveKineticBodies, gatherKineticContacts, and the sleep tick to that policy so the active set shrinks to bodies that are moving, being steered, or in unstable contact — not everyone who happens to be touching someone. Success for the stress test: after things settle, frame cost drops because most of the cavern is dormant, not because we skipped a check in one hot loop.

PR 2 — Contact pipeline refactor. With a trustworthy active set, rebuild the kinetic collision pass as explicit stages: snapshot broadphase bounds once per body per substep, emit candidate pairs once (spatial neighbors + id ordering + pair policy), then narrow-phase and solve only on that list. gatherKineticContacts stops re-querying bounds and re-testing overlap for the same pair in multiple places; contact buffers, warm-start, and iteration count all operate on one canonical pair stream. This PR is mostly structure, not tuning — the goal is a pipeline you can reason about and extend (new prop types, new pair filters, fewer substeps when stable) without sprouting one-off perf branches. Success: same sim behavior on small scenes, but the stress test profile shifts from “death by pair enumeration” to a proportional cost on actually awake contacts.

PR 3 — Shape-tier narrow phase. Most stress-test props are circles; wedges, boxes, and glass are the SAT-heavy tail. Split contact creation and resolution by shape tier — circle–circle, circle–poly, poly–poly — with circles using a dedicated path from broadphase through contact generation (no SAT dispatch, no polygon projection for ball piles). Polygons keep the current SAT path unchanged. That gives you a default fast lane for the common case and a clear place to hang future tiers (capsules, compounds, static kinematic pairs) as the engine grows. Success for the endgame: load the cavern stress demo, let it settle, pan around and issue nav — most frames stay smooth because awake work scales with moving bodies and shape complexity, not raw prop count.
