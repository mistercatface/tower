PR 2 — Autonomous goal-seek + consume + grow loop (engine-facing, not game-launch-facing)
Introduce a small Libraries/Sandbox/autosim/ layer (not logic in preview.js or a one-off in sandboxStartScene): a createGoalSeekAutosim(state, config) tick driver that (1) resolves the steering prop via resolveGroundNavSteeringProp / chain head, (2) calls the registered HPA behavior’s setMoveTarget toward a goal prop’s world position, (3) on proximity fires a consume callback. The snake-specific callback lives in snakeAutosim.js: despawn goal, addChainLink a new tail segment at configured spacing, respawn goal on a random open cavern cell excluding snake-occupied cells, re-issue nav. Wire the autosim through the existing sandbox controller tick path (same mount as other behaviors) behind a config flag so editor sessions stay untouched until opted in. Add tests/snakeAutosim.test.js with mocked state: eat once → constraint count +1, goal position changes, head gets a new move target — this doubles as the physics.md “recommended next” integration test for repeated link creation and documents tail/wall clipping behavior without needing a full sim.
