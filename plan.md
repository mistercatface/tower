PR 1 — Selection UX. Shift+drag on the canvas draws a marquee and selects props whose bounds are fully inside the box (strict containment, not center/circle overlap). The left sidebar gains a Selection panel on par with Scene: filter chips (All, Shapes, Nav, Gen, Rooms) driven by the same sandbox.tags as the prop palette, a live list of what’s selected, and bulk actions like “select all {filter} in scene” and “filter selection to {filter}”. Marquee and those filters share one tag-matching helper, so adding a category later is just tagging assets — no new selection code paths.

PR 2 — Mass ground nav. Right-click on the canvas opens a context menu when you have a prop selection; Move with HPA and Move with Flow send every selected prop tagged nav to that world point in one shot, auto-stamping the behavior so you don’t switch each prop off drag-launch first. A single issueGroundNavToSelection command replaces today’s map-wide P+click logic and scopes to the current selection only. Flow and direct ground nav get the same per-prop session + tickWorld pattern HPA already has, so multiple agents can path concurrently — the main engine win beyond the editor feature.

PR 3 — Cleanup and polish. Remove the P key + click mass-HPA path entirely (pKeyHeld, issueMassHpaGroundNav, the interact-tool hook). The Selected panel and context menu share the same mass-nav actions; path debug overlays draw for all selected props with an active nav run, not just the primary one. Optional ctrl+click toggle for additive selection rounds it out. End state: shift+drag to box-select nav balls, right-click destination to path them all — no hidden modifier, no manual behavior switching, no selecting the whole map by accident.
