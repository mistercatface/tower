# Sprite cache API — call-site objects & parameter soup

## The issue

`getOrBakePropSprite({ prop, px, py, renderKey, draw, … })` and `getOrBakeOverlaySprite({ worldX, worldY, px, py, … })` (plus wrapper opts on `drawCachedPropSprite` / `drawCachedOverlayGlyph`) allocate a fresh options object on every blit; V8 may elide that literal on a monomorphic hot path, but it is not guaranteed, and the shape reads like scratch pushed onto callers. Worse, the call-site object is usually not the dominant cost: every cache lookup still runs `buildPropSpriteKey` / `buildOverlaySpriteKey`, which concatenates a template string and calls `quantizeView`, which returns a new `{ dx, dy, keyDx, keyDy }` object — so steady-state drawing pays string + small-object churn on every hit even when bake never runs. The options-object pattern also forces a wide, awkward surface (7–9 named fields per call) duplicated across `PropRenderer`, grid-stamp draw, and overlay glyphs.

## Ideal refactor

Collapse to two positional entry points — `drawCachedPropSprite(ctx, drawPass, prop, renderKey, draw, animFrame?, zoom?, modifier?)` and `drawCachedOverlayGlyph(ctx, drawPass, worldX, worldY, renderKey, customKey, worldSpan, draw, zoom?)` — where `drawPass` is a per-frame struct already owned by the renderer (`{ px, py, zoom, propRecipes? }`, same idea as `WorldSceneRenderer.propDrawContext`), so leaf call sites pass one context plus entity-specific args instead of repeating `px`/`py`/`zoom`/`propRecipes` on every prop and belt cell. Keep `getOrBake*` internal to `QuantizedSpriteCache.js` with positional scalars only; callers never touch the bake layer. Replace string cache keys with a packed numeric identity (stable prop/overlay ids + quantized view bucket + anim/zoom/custom slots) and make `quantizeView` write into module-private scalars or return a single packed int — that removes the per-lookup string and object allocs that matter on cache hits, not just the literal at the call site. Net: fewer parameters at call sites, no options objects, no caller-owned scratch, and the hot path stops building throwaway key material every frame.
