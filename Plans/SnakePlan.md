PR 1 — Introduce SnakeInstance and make it the owner (shim only, zero behavior change). Add SnakeInstance.js with a small class: headId, spawnGroupId, autosim, lifecycle (alive | inert | dead), memberIds cache, and methods start(), stopSteering(), retireAllSegments(state), syncMembersFromGraph(state). Add SnakeGame (or extend snakeGame state) with instancesByHeadId / instancesById instead of the loose trio of registry.aliveByHeadId, autosimsByHeadId, and free functions in snakeCombat.js. Wire setupSnakeGame spawn loop to new SnakeInstance(...).start() and make killSnake / splitSnakeAtStruckSegment / resolveSnakeCombatFromContacts delegate to instance methods — but keep syncSnakeGameLifecycle running unchanged behind those delegates so rolling heads cannot regress while you migrate. Tests: existing snakeSplit, snakeMinLengthDeath, snakeStrikerSplit must pass untouched; add one test that instance stopSteering() clears \_groundRollDrive on all members.

PR 2 — Single mutation API + steering lease (this is where desync dies). Every graph/registry change goes through SnakeInstance.splitAt(segmentId), .die(), .severInertTail(...) — each method runs in fixed order: cut links → update members → stopSteering() → retireAllSegments() → update lifecycle maps. No other file calls retireSnakeSegmentsFromNav, autosim.stop(), or markSnakeDead directly. Add a steering lease on the head prop only: \_snakeSteering = { instanceId, epoch } set in start(), cleared in stopSteering(). Gate steerRollToward / applyGroundRollDrive: if prop has \_snakeSteering and it doesn't match a live instance+epoch, no-op (don't set drive). Combat/striker resolve memberId → SnakeInstance via instance lookup, not buildAliveSnakeMemberHeadMap rebuilt from graph every contact. syncSnakeGameLifecycle shrinks to a dev-only assert or thin wrapper calling instance.validate() — still present but should become a no-op in tests.

PR 3 — Delete the guard rails once leases enforce invariants. Remove syncSnakeGameLifecycle, sweepOrphanSnakeChains, isSnakeHeadPhysicallyAttached, and collectSnakeSpawnGroupMemberIds world scan — instance memberIds + spawn group id captured at birth make orphan sweeps redundant because nothing can retire/kill/steer outside the instance API. Autosim tick only runs on instance.isSteerable() (alive + head live + chain head + min length); invalid instances call instance.die() inline instead of a global janitor pass. Keep the two real bug fixes permanently: unique spawn group ids (linkedBallChain:${props[0].id}) and larger-head-only combat strikes — but encoded inside instance/combat methods, not scattered helpers. Exit criteria: same 15+ snake tests green, plus new tests for “kill mid-physics same frame”, “split then immediate striker hit”, “stale autosim map entry”, all asserting \_groundRollDrive undefined and no autosim tick after death without calling sync.

It didn't exist before because snakes were modeled as props + maps: each feature (autosim, combat, striker, starvation, spawn groups) patched shared global state (aliveByHeadId, kinetic graph, \_groundRollDrive) independently instead of one object whose constructor/destructor owns the invariant “only my head may be steered, and only while I'm alive.” The guard rails are the symptom of that missing object — the three PRs above install the object first, enforce it second, then delete the symptom third.
