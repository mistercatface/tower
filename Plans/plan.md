PR 1 — Libraries/Color + pre-surgery catalog + bake plumbing
Introduce Libraries/Color as the single home for hex/rgb/hsl, palette hue-shift, brightness scaling, and picker helpers — migrate propTint.js (then delete it) and any obvious duplicates (poolBalls shadeHex, procedural clampByte callers where trivial). Pre-surgery: add canonical shape assets with neutral base coats — e.g. one ball (grey sphere panels, shared physics with today’s balls) and polygon archetypes hex_block / tri_wedge / block (simple extruded greys, no plankTs/topCross in the live draw path; move decorative box recipes to Libraries/deprecated/ if kept at all). Old ids (blue_ball, orange_ball, steel_ball, box_2x4, …) stay registered temporarily as thin aliases (same spawn, or spawn that creates the canonical type + default tint preset) so nothing breaks mid-flight. Instance visualOverride { tint, brightness } on placed props, snapshot field, cache key in getBaseSpriteCacheKey, resolve only at bake via Color → base coat. Snake uses ball + override. Outcome: one color stack, one bake path, canonical assets exist, soup not yet removed from the palette grid.

Palette grid can still list old names briefly, but primary rows are Ball + Block (+ special gameplay props: snake_head, goal_orb, flipper, …). Tests: override round-trip, radius/footprint change rebakes, spawn defaults apply on place.
