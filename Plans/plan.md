## INITIAL PLAN

Your profile is telling a slightly different story than “physics is the problem.” The snake AI/perception stack is eating the frame first: findNearestVisibleSnakeGoal (~16%), queryGridCellVision + collectVisibleGridCells (~24% combined), plus cardinalLegOpen on the LOS path. That’s roughly 40% of wall time before you even get to constraints. Within physics, the meaningful chunk is constraint work: solveDistanceConstraintVelocity (~6%) plus the island loop in kineticConstraintSolver (~9%), which is mostly distance joints + link-capsule-vs-wall projection on every awake snake chain. So a pure physics PR can move the needle, but probably on the order of 10–15%, not half the frame — unless you pair it with the vision/goal fixes.

The best physics-only next step is still the planned link-capsule wall culling work in kineticConstraintSolver.js: reuse the per-island walls[] buffer (partially there already), AABB-filter wall candidates per link before getLinkCapsuleSegmentPenetration, and skip asleep / near-static links harder. That’s gameplay-neutral, already scoped in your old plan notes, covered by linkCapsuleWall.test.js, and it advances physics.md Tier 6 (chain vs walls), Tier 8 (performance), and the “snake as physics proving ground” item — plus it buys headroom before more wall-damage traffic makes constraint passes worse. A second physics lever with smaller but real payoff is constraint velocity early-out per link (skip pairs already at rest length within epsilon, cache last-solve impulse per constraint id) — same Tier 8 lane, touches kineticBodySlab/SoA direction eventually but doesn’t require SIMD yet.

If you want something that hits multiple roadmap pillars at once, the highest-ROI bundle isn’t physics — it’s a “many-snake perception” PR: spatial index for goal orbs (kills the O(snakes × goals) collectSnakeGoalProps scan), plus incremental/throttled FOV sync (you already have brainSyncOffScreenInterval — extend it with a shared goal visibility cache or cell-bucketed goals). That one PR advances AI.md (perception + utility-scoring prep), pathfinding.md (agent scale), ROADMAP AI pillar, and indirectly physics.md Tier 9 (snakePerfBudget) because you can actually run 50 snakes in the budget test. Belt-side damage and LosShadow/shadowProjection are worthwhile endgame goals, but they’re the opposite of perf wins right now: belt damage pulls in Mazes.md D-belt-safe post-process, nav resync, and destruction plumbing (physics.md belts at ~40%); shadows are rendering.md lighting debt and add draw cost. I’d treat those as feature PRs after the scale pass, not as the next optimization round.

## PR ROUNDS

PR 1 — Link-capsule wall culling (physics, Tier 6 + Tier 8)
Hoist and reuse the per-island walls[] buffer in projectIslandLinkCapsulesAgainstWalls so each island gathers wall candidates once, then tighten broadphase: each link capsule AABB filters the island wall list before getLinkCapsuleSegmentPenetration. Harder early-outs for asleep islands (already partially there), pairs where neither body is kinematically active and the capsule doesn’t overlap any gathered wall, and links below a velocity threshold. No formula or iteration-count changes — same corrections, fewer narrow-phase calls. Ship with linkCapsuleWall.test.js plus a quick extension to snakePerfBudget.test.js or a micro-benchmark on N chains in a corridor maze. Success metric: meaningful drop in projectIslandLinkCapsulesAgainstWalls / kineticConstraintSolver self time with identical overlap fixtures.

PR 2 — Many-snake perception scale (AI + pathfinding + Tier 9 benchmark)
Replace the O(snakes × goals) collectSnakeGoalProps scan inside findNearestVisibleSnakeGoal with a maintained goal index — simplest win is a live Set/Map of goal prop ids updated on spawn/eat/remove, optionally cell-bucketed by obstacleGrid.worldToGrid so vision only considers goals in cone range cells. Layer throttled FOV on top of existing brainSyncOffScreenInterval: off-screen snakes already sync every N ticks; on-screen snakes get queryGridCellVision only when heading/range/cell changes materially, with a per-tick shared LOS cache keyed by (observerCell, targetCell, wallRevision) so multiple snakes in the same corridor don’t re-walk Bresenham. cardinalLegOpen stays on the nav context — don’t rebuild it per query. Tests: goal spawn/eat index consistency, visibility parity vs today on a fixed fixture, snakePerfBudget bumped to 50 snakes within ceiling. This is the PR that moves physics.md Tier 9 benchmark from aspirational to honest.

PR 3 — Constraint velocity early-out (physics polish, Tier 8 → slab direction)
After PR 1–2, profile again; if solveDistanceConstraintVelocity is still visible, add per-constraint skip logic in the velocity PGS loop: if |dist - restLength| < positionEpsilon and relative normal velocity is below threshold, skip the impulse solve for that link this iteration. Optionally cache lastImpulse per constraint id (slab slot or link key) to warm-skip consecutive near-zero solves — same pattern as contact warm-start, decayed each outer iter. Guard with linkCapsuleWall.test.js-style fixtures plus a snake-chain wiggle test so chains don’t go slack under load. Small diff, but it’s the right stepping stone toward slab-local state and eventual SoA without touching SIMD yet. Land only if PR 1’s profile shows constraint velocity still >~3% self time; otherwise fold into a later physics pass.

What this unlocks later (not in these three): belt-side damage and LosShadow/shadows stay out of this trilogy — they’re feature debt, not scale wins. After these three, you’ll have a honest 50-snake budget, a cleaner physics profile, and a perception layer ready for utility scoring / flee-pursue without another architectural pass.
