PR 1 — Active set + constraint pass hygiene
Your profile is dominated by solveDistanceConstraint and the wake path through activateKineticBody / \_ensureActive: every link solve does a linear scan of the active array and re-wakes the whole island, multiplied by motion substeps × kinetic iterations × constraint iterations × ~60 links in the 30-snake cavern. This PR fixes the structural waste without changing sim behavior. Replace \_ensureActive's O(n) scan with O(1) membership (slot index on the prop, swap-remove from \_activeKineticBodies). Batch pending wakes: constraint and contact inner loops mark bodies dirty, one flush per constraint iteration (or per substep) instead of per-link activateKineticBody with full island walk.

Second change: stop re-gathering constraints on every inner loop. collisionPipeline calls resolveKineticConstraintPass four times per substep; each call rescans state.sandbox.kineticConstraints. Gather once per substep into the existing buffer, then iterate solve only. Same topology within a substep — no gameplay change, fewer registry walks and less allocator churn on the hot path.

Acceptance: headless benchmark — 30 three-segment snakes, N ticks, wall-clock or iteration count drops measurably vs baseline; unit tests that active membership stays correct through wake/sleep/evict; /?game=snake cavern visibly smoother in Chrome perf (constraint + activate slices down, not just hoped). No new collision settings exposed yet — pure engine hygiene. Unlocks Tier 8 “profile-friendly pipeline” and sets up everything in PR 2–3.

PR 2 — Chain iteration budget + island rebuild GC
After PR 1, constraint solve time is still linear in links × iterations even when chains are nearly at rest — snake tails sitting in corners keep getting full 4×4 solve blocks because links never declare “good enough.” Add per-substep early-out on the constraint pass: track max distance error across the buffer; if below ε and linked bodies are below velocity thresholds, skip remaining constraint iterations for that substep. Optionally skip contact gather for island members that share a root and none are moving (internal pairs already skipped; this stops paying iteration tax on sleeping chains). Tune ε against chainVsWallGrowth and snake growth so moving heads still get full solve.

Parallel GC win: buildKineticIslands allocates fresh Maps every frame in runKineticPhysics. Rebuild islands only when the constraint list is dirty (link add/remove/grow — snake growChainSegment, resyncChainLinkRestLengths) or membership changes; otherwise reuse peer arrays from last frame. That targets the ~6% Major GC slice without touching gameplay.

Wire a snake collision profile via applyGameCollisionSettings in setupSnakeGame — not global defaults. Slightly lower kineticConstraints.iterations or kineticIterations for the multi-snake cavern if early-out doesn't alone hit budget; document the profile in Config/games/snake.js. Acceptance is snake-specific: 30 snakes + 15 goals at default config holds target FPS (you pick the number — 60 on your machine) while eating/growing doesn't stutter. Headless test asserts early-out fires on a settled chain fixture and that island dirty-flag skips rebuild when constraints unchanged.
