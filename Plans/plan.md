Pass 1: Fast-Out for Kinetic Sleep Wake Traversals The wakeKineticBody function currently takes a massive 70.5ms because it repeatedly iterates over a body's entire \_kineticIslandPeers and \_kineticLinkNeighbors arrays every time a constraint touches the body—even if the body and its peers are already fully awake. In dense scenes, this results in tens of thousands of redundant array traversals per frame. We will add a simple fast-path at the very top of the function (if (!entity.isSleeping && entity.\_sleepFrames === 0) return;). This instantly short-circuits the traversal, completely eliminating this 70.5ms block from the profile while ensuring bodies still correctly wake up when a sleep state actually needs to be broken.

Pass 2: Precompute Distance Constraint Jacobians in the Velocity Solver The solveDistanceConstraintVelocity function consumes 250.6ms because it dynamically recomputes world anchors, distance vectors, normals, and effective mass denominators (k) inside the high-frequency solver iteration loop. Since body positions and angles do not change during velocity-level iterations, these geometric values are mathematically constant. We will expand the kineticConstraintSlab structure to store the precomputed normal (nx, ny), rotational leverages (rAn, rBn), constraint mass (k), and positional error. By calculating these once during warmStartKineticConstraintSlab (or a dedicated pre-step) and replacing the math with flat array lookups inside the iteration loop, we will dramatically slash the overhead per iteration, likely cutting the solver time in half.

Pass 3: Eliminate O(N^2) Wall Candidate Gathering The broadphase WorldObstacleGrid queries (140.5ms) are bottlenecked by how wall candidates are accumulated for kinetic islands. gatherIslandLinkWallCandidates currently loops over every body in an island, queries the grid for nearby wall proxies, and then feeds them into mergeWallCandidatesInto which does an O(N^2) linear scan to deduplicate them. Even worse, because the grid proxies are issued sequentially from a pool on every query, the === identity check fails, causing the islandLinkWallCandidates array to balloon with duplicate proxies. We will optimize this by calculating a single unified bounding box (AABB) that covers the entire kinetic island, and then perform just one appendStaticWallProxiesNearWorld query against the grid for that entire region. This skips the array merging and proxy duplication entirely, collapsing the broadphase overhead.
